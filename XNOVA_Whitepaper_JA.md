## 1. 概要

XNOVA は、DePIN（分散型物理インフラネットワーク）の概念に着想を得たデフレ型マイニング・エコシステムであり、BNB Chain 上に構築され、現在 Base および Polygon への展開が進行中です。報酬を支払うために新規トークンの発行に依存する従来のイールドファーミングやステーキングのモデルとは異なり、XNOVA のマイニングメカニズムは、ユーザーが購入する「Boost（ブースト）」と NFT によって完全に資金供給されています。Boost の有効化や NFT のミントに費やされたすべての BNB は、PancakeSwap 上の永久的かつ回収不可能な流動性へと直接変換され、生成された LP トークンは即座にバーン（焼却）されます。これにより、チームを含む誰によっても決して引き出すことのできない、常に増加し続ける流動性の下限が生まれます。さらに、ローンチ時に総供給量の 50% がすでにバーンされた、上限のある再発行不可能なトークン供給と組み合わされています。

XNOVA は単なるトークンではありません。マルチチェーン DEX アグリゲーター（xNovaSwap）、フェアローンチ型ミームトークン発行プラットフォーム（NOVAFUN）、NFT マーケットプレイス（Kovyn）、ブラウザ拡張機能型ウォレット（XNEO Wallet）、Web3 コミュニティチャットプラットフォーム（XNova Chat）、トークン保有者限定の AI コンソール（XNOVA AI）、そして独自のエクスプローラーとフォーセットを備えた独自構築の EVM 互換ブロックチェーン（XNova Chain）を含む、より広範なエコシステムの基盤なのです。

## 2. はじめに

Web3 のマイニングおよび「GameFi」分野は、持続不可能なトークン発行によって資金供給される高利回りを謳うプロジェクトで飽和しており、初期参加者が後発参加者を犠牲にして利益を得る構造になっています。こうしたプロジェクトの大半には、次の 3 つの共通する失敗要因があります。

- **インフレ的な報酬発行** — 時間の経過とともに保有者の価値を希薄化させる
- **ロックされていない、または一時的にのみロックされた流動性** — 最終的にはチームによって引き出される可能性がある
- **不透明なトークノミクス** — オンチェーンで検証できない

XNOVA は、初日からこれら 3 つの失敗要因すべてを排除することを目的として、特別に設計されました。

## 3. 課題

現在の Web3 における「マイニング」または「Boost」プラットフォームの大半は、以下 2 つの欠陥あるモデルのいずれかで運営されています。

1. **ポンジ式発行モデル** — 新規ユーザーの入金が既存ユーザーへの利回り支払いに充てられ、実質的な価値創造がないまま、やがてモデルは崩壊します。
2. **ロックされた（バーンされていない）流動性** — チームは LP トークンを一定期間（例：100 日、1 年、あるいは 100 年）ロックしますが、ロックは永久的なものではありません。ロックは期限切れになる可能性があり、不正に延長される可能性があり、あるいはロック用コントラクト自体が侵害された場合や、それがチームに管理されたプロキシである場合には、迂回される可能性もあります。

いずれのモデルも、チームまたは第三者のロックサービスに対する信頼に依存しています。XNOVA は、この依存関係を完全に排除します。

## 4. XNOVA のソリューション

XNOVA は、「ロックされた」流動性を「バーンされた」流動性に置き換えます。これは本質的にはるかに強固な保証です。

| モデル | 流動性は将来引き出され得るか？ |
|---|---|
| ロックされていない流動性 | はい、いつでも |
| 期間限定でロックされた流動性（例：100 年） | はい、ロック期間の終了時、またはロック用コントラクトが侵害された場合 |
| **バーンされた LP トークン（XNOVA モデル）** | **いいえ — いかなる状況下でも決して起こりません** |

LP トークンがバーンアドレス（秘密鍵が存在しないことが分かっているウォレット、例：`0x000...dEaD` または同等のヌルアドレス）に送信されると、それらは永久的かつ数学的に回収不可能になります。これは方針や約束ではなく、トークン自体を保護しているのと同じブロックチェーンによって強制される暗号学的保証です。

ユーザーがマイニング用の Boost を購入するか、Supporter NFT NFT をミントするたびに、受け取った BNB の 100% が PancakeSwap V2 上の XNOVA/BNB 流動性に変換され、生成された LP トークンはバーンされます。これは以下を意味します。

- 流動性プールの深さは増加し続けるのみで、決して縮小しません。
- 悪意の有無にかかわらず、その流動性が取り除かれるシナリオは一切存在しません。
- プラットフォーム上での購入一つひとつが、トークンの価格の下限を直接的かつ永続的に強化します。

## 5. マイニングの仕組み

```
1. BNB で Boost を購入
   ユーザーは BNB で Boost パッケージを購入することで
   マイニングを有効化します。

2. XNOVA トークンをマイニング
   Boost は投資額に応じて、時間の経過とともに
   XNOVA 報酬を生み出します。

3. BNB → 流動性
   各 Boost 購入の BNB の 100% が、PancakeSwap V2 上の
   XNOVA/BNB 流動性に変換されます。

4. LP トークンをバーン
   その流動性から生成された LP トークンは、即座かつ
   永久的にバーンされます — ロックでもベスティングでも
   ありません。永遠に消滅します。

5. デフレ型供給
   ローンチ時の供給量 50% のバーンと組み合わさることで、
   XNOVA の流通量と流動性は、常に一方向にのみ動きます:
   減少し、より永久的に確保された状態へと向かいます。
```

マイニング報酬は、総供給量の 20% を占める専用の「Mining Rewards」配分から、公式マイニングスマートコントラクトを通じて、ユーザーの有効な Boost の規模と期間に基づき、時間の経過とともに分配されます。

## 7. バーンメカニズム

XNOVA の核心的な信頼の基盤は、購入のたびに発生する反復的な流動性バーンです。

1. ユーザーが BNB で支払い、Boost を購入するか、Supporter NFT NFT をミントします。
2. その BNB の 100% が XNOVA とペアリングされ、PancakeSwap V2 上の XNOVA/BNB プールに流動性として追加されます。
3. その入金から生成された LP トークンはバーンアドレスへ送信されます。
4. この LP ポジションは、チーム、開発者、いかなる第三者によっても決して引き出すことができません — 数学的にアクセス不可能です。

これは、期間限定の流動性ロックとは異なり、それよりもはるかに強力です。ロックには期限があるという点で「約束」に過ぎません。バーンには期限も、所有者も、鍵も存在しません。

*注：バーンの正確な実行頻度（トランザクションごとか、バッチ処理か）、およびバーンがスマートコントラクトのロジックによって自動的に実行されるのか、あるいは公開的に検証可能な手動トランザクションによって実行されるのかについては、透明性を持って開示されるべきであり、完全な監査可能性を維持するために、発生の都度、BscScan 上のリアルタイムのバーン取引リンクを添付すべきです。*

## 9. 技術アーキテクチャ

- **トークン規格：** BEP-20（BNB Chain）。XNova Chain やその他の EVM ネットワークへのブリッジインフラを通じて、クロスチェーン展開を行っています。
- **マイニングコントラクト：** アクティブな Boost 保有者に対し、Boost の規模と期間に応じて比例配分で XNOVA 報酬を分配します。
- **流動性コントラクトのフロー：** Boost/NFT の支払い → PancakeSwap V2 での自動 LP ペアリング → LP トークンのバーン。
- **XNova Chain ブリッジ：** BNB Chain と XNova Chain の間の Lock-and-Mint 方式のブリッジアーキテクチャで、リレーサービスとリプレイ攻撃対策を備えています。
- **NFT レイヤー（Supporter NFT NFT）：** シードベースのカラーパレットによる完全オンチェーンの SVG 生成により、NFT のメタデータがオフチェーンで失われたり改ざんされたりすることがありません。

## 10. セキュリティと信頼性

- **ミント機能なし** — 総供給量は不変であり、減少することしかできません。
- **流動性引き出し経路なし** — LP トークンはバーンされており、いかなるウォレットやロック用コントラクトにも保有されていません。
- **オンチェーンでの検証可能性** — すべてのバーン、流動性の追加、マイニング分配は BscScan を通じて検証可能です。
- **チーム配分は売却税のみを原資とする** — チームは事前のトークン配分を一切受け取らず、早期の利益抽出ではなくエコシステムの長期的な成長にインセンティブを合わせています。

*推奨される追加事項：主要な取引所上場に向けた大きな取り組みの前に、独立したスマートコントラクト監査レポートを委託し、本セクションにリンクを掲載することで、さらに信頼性を強化すべきです。*

## 13. リスク開示

- 暗号資産および DeFi 資産は非常に変動性が高く、トークンの価値は大幅に下落する可能性があります。
- スマートコントラクトは注意深く設計されているとはいえ、予期しない脆弱性を含んでいる可能性があります。ユーザーは失っても構わない資金のみを投じるべきです。
- トークン、マイニング報酬、NFT に対する規制上の取り扱いは法域によって異なり、変更される可能性があります。
- エコシステムの過去の成長やこれまでのバーンイベントは、将来のパフォーマンスを保証するものではありません。
